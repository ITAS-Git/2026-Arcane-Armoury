## Initial Version of the Project

Originally we wanted to create the project to be used on a laptop and have it display onto a TV, projector, or anything larger in scale. This idea was later abandoned and the project ended up being a Raspberry Pi connected to dual monitors, as using a laptop would cause problems later on and securing a large TV or projector would also be hard to acquire for the purposes of a single project.

## Potential Future Improvements

Things we later thought to add after project completion that would improve the overall project should we choose to continue it after semester's end:

- **Power Switch for the Raspberry Pi** — Currently, the system requires manual steps to start up. We intended to implement a cleaner boot process or physical switch to make powering on and off more seamless for users at the table.
- **Background Music** — We had planned to integrate ambient background music into the player-facing display to enhance immersion during sessions. The feature was scoped out due to time constraints and was not implemented in the final build.

## Lessons Learned

This project taught us valuable lessons, including:

- **Communication** is far more critical than it first appears in a larger, multi-component project. As the system grew more complex, clear and timely communication between team members became essential to avoiding duplicated effort and cascading bugs.
- **Git and GitHub** are invaluable tools. Using issues, commit history, and the Kanban board kept the team aligned and gave everyone visibility into what was being worked on at any time.
- **Sticking to the defined scope** is harder than expected, especially when new ideas or improvements become apparent mid-project. Learning to evaluate whether a new idea is worth the scope risk is a skill we all developed over the course of this project.

## Good Points

- **The Bluetooth Remote** — The ESP32 BLE remote successfully communicated with the Raspberry Pi, giving players a physical way to interact with the system without breaking immersion at the table.
- **3D Printed Shell Casing** — The custom-printed enclosure gave the remote a polished, professional look and protected the breadboard components during use.

## Drawbacks

- **Startup Process** — The Raspberry Pi requires manual steps to bring the system online. A cleaner boot experience would make the system more accessible for non-technical users at the table.
